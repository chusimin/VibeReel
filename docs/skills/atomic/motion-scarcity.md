# 稀缺动效 = 结构性重量

## 核心洞察

VibeReel 现在生成的视频"看起来乱"，**根本原因不是动效不够多，是每一镜都用了不同的动效**——观众的眼睛无法建立"这条视频的动效基因"。

**参考视频反过来做**：Aftermagics × Datalyr 全片 30+ 个动效实例里：

| 动词 | 次数 | 占比 |
|---|---|---|
| fade / fade-in | 8 | 27% |
| slide | 7 | 23% |
| typewriter | 5 | 17% |
| **前 3 合计** | **20** | **67%** |
| mask / mask-reveal | 4 | 13% |（只在关键转场）
| scale | 2 | 7% |（只给 logo）
| 其他 | 4 | 13% |

**规律**：**主动效三件套占 60-80%**，稀缺动效只用在结构性时刻。

## 规则

一条 showreel 必须**先定"动效基因"**，然后按下面分配：

- **主动效基因**：3 个动词，覆盖 60-80% 分镜
- **点缀动效**：2-3 个动词，覆盖 15-25% 分镜
- **稀缺动效**：1-2 个动词，只在**关键时刻**用（每种全片 ≤ 3 次）

**"关键时刻"定义**：
- 段与段之间的转场（不是镜与镜之间）
- 视觉高潮 / drop 镜
- Logo 出现镜
- CTA 出现镜

## 反例

- ❌ 8 个镜用 8 种不同动效 → "花哨"感，观众无法记忆
- ❌ mask reveal 每镜都用 → 失去转场感
- ❌ scale 到处用 → logo 出现时没有"结构性重量"
- ❌ 没有主动效基因，全靠 renderer 默认 → 拼贴感

## Agent 使用指令

> 生成分镜前，先给出这条视频的"动效基因"：
> ```json
> {
>   "primary": ["fade", "slide", "typewriter"],
>   "secondary": ["blur", "color-change"],
>   "scarce": ["mask-reveal", "scale"]
> }
> ```
> 然后每一镜必须声明用了哪个动词，且必须来自这三组。
> scarce 动词每种全片使用不能超过 3 次。

## 待验证

- [ ] "3 个主动效"数字是否稳定？
- [ ] 不同类型 showreel（品牌 vs 个人 motion）的动效基因组合是否有典型 pattern？
