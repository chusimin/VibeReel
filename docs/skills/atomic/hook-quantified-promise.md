# 量化承诺型 hook

## 观察

Aftermagics × Datalyr 前 3.2s：
- 0-1.9s：**空拍铺垫**（几乎无画面变化，音乐建立预期）
- 1.9-3.2s：大字 **"in under three minutes"** 淡入

**这个 hook 不是"炸场画面"，是"轻声承诺"。** 关键是**文案里的量化数字**。

## 为什么有效

1. **"三分钟"是具体的、可验证的承诺** —— 观众立即知道"接下来我要看到 3 分钟内能做完的东西"
2. **前 1.9s 的空拍是必需的** —— 没有它，"三分钟"承诺没有分量
3. **文案短到能瞬间读完**（4 词），停留时间足够（1.3s）

## 规则

产品型 showreel 的 hook 优先用"量化承诺"，格式：

```
[数字/时间/规模] + [动作/结果]

good: "in under 3 minutes"
good: "10× faster than before"
good: "1 click, zero config"
good: "从 0 到 1，30 天"

bad: "让工作更高效" (无量化)
bad: "改变你的方式" (无量化 + 空泛)
bad: "The future of X" (陈词滥调)
```

**结构**：
- 空拍 1.5-2s（音乐进入，无画面变化）
- 量化承诺文字 1-1.5s（居中大字，1-2 关键词变色强调）
- 承诺文字消失后立刻进入"演示"

## Agent 使用指令

> 如果视频类型是 "showreel-product" 且用户提供了产品描述：
> - 分析用户输入，提取一个"可量化的产品承诺"作为第一段文字
> - 第一镜设计为"空拍 + 承诺文字"，durationSec 2.5-3.5s，其中前 1.5s 无文字
> - 承诺文字里的核心数字用 accent 色强调

## 反例

- ❌ 无空拍，直接文字上来 → 观众没准备好
- ❌ 承诺文字超过 8 词 → 读不完
- ❌ 用无量化的空泛口号 → 无 hook 效果
