# Showreel 反例库

> Agent 光知道"什么是好"还不够，还得知道"边界在哪"。**每个反例都是"如果没做 X，成片就会怎样丑"的反向陈述**。
>
> **本文档随每次生成失败/被打回自动补充**。

## 反例分类

### R1 · 节奏类反例

**R1.1 · 拍不上点**
- 表现：分镜切换时间与音乐节拍无关，"啪嗒啪嗒"
- 原因：`durationSec` 凭空定，没参考 BPM
- 修法：见 [beat-vs-onset](../atomic/beat-vs-onset.md)
- 严重程度：🔴 致命（这是"业余感"的第一来源）

**R1.2 · 前 3 秒就炸场**
- 表现：视频一开始就是快切+大特效
- 原因：没有 hook 缓冲期
- 修法：见 [hook-quantified-promise](../atomic/hook-quantified-promise.md)（前 1.5-2s 必须空拍）
- 严重程度：🟡 严重

**R1.3 · 末尾 CTA 还在动**
- 表现：URL / 按钮出现后立即消失或还在动画中，视频就结束了
- 原因：没有"静止 1.5s"收束
- 修法：见 [brand-outro-3step](../atomic/brand-outro-3step.md)
- 严重程度：🔴 致命（用户记不住网址 = CTA 无效）

### R2 · 视觉类反例

**R2.1 · 颜色 5 种以上**
- 表现：每一镜换一种主色，"彩虹感"
- 修法：见 [color-92-6-2](../atomic/color-92-6-2.md)（主色只 2-3 组）
- 严重程度：🔴 致命（拼贴感来源之一）

**R2.2 · 视觉密度太高**
- 表现：屏幕塞满 6+ 元素
- 修法：见 [density-arc](../atomic/density-arc.md)（禁止"密"级别）
- 严重程度：🔴 致命

**R2.3 · 没有视觉母题**
- 表现：每镜元素形状、结构都不一样
- 修法：见 [broken-pattern-highlight](../atomic/broken-pattern-highlight.md)（必须有一个母题）
- 严重程度：🟡 严重

**R2.4 · 母题被打破多次**
- 表现：整片好几次"惊喜时刻"，观众麻木
- 修法：全片只允许 1 次 drop，位置在 60-75% 时长
- 严重程度：🟡 严重

### R3 · 动效类反例

**R3.1 · 每镜用不同动效**
- 表现：8 个镜 8 种动效，"花哨"
- 修法：见 [motion-scarcity](../atomic/motion-scarcity.md)（主动效 3 件套占 60-80%）
- 严重程度：🔴 致命

**R3.2 · 稀缺动效被滥用**
- 表现：mask-reveal 每镜都用，scale 到处飞
- 修法：mask/scale 每种全片 ≤ 3 次，只在关键时刻用
- 严重程度：🟡 严重

**R3.3 · 用文字花式动画呈现 URL**
- 表现：CTA 用 kinetic text / 变形 / 旋转 / 打字机后立即消失
- 修法：CTA 必须静止呈现，让用户"记住"
- 严重程度：🔴 致命

### R4 · 文案类反例

**R4.1 · 空泛承诺**
- 表现："让工作更高效"、"改变你的方式"、"The future of X"
- 修法：hook 必须是量化承诺，见 [hook-quantified-promise](../atomic/hook-quantified-promise.md)
- 严重程度：🟡 严重

**R4.2 · 文字太多来不及读**
- 表现：一屏 15+ 词，停留 <1.5s
- 修法：每屏文字 ≤ 8 词，停留 ≥ 1.5s
- 严重程度：🟡 严重

**R4.3 · 用户输入原样出现在视频里**
- 表现：用户说"我做了一个数据分析工具"，视频里就出现这句话
- 修法：Agent 必须把用户输入**改写成 hook 文案**（量化 + 具体动词）
- 严重程度：🔴 致命（这是 VibeReel 现在的问题之一）

### R5 · 结构类反例

**R5.1 · 卖点罗列型**
- 表现：视频里"我们有 A / 我们有 B / 我们有 C"平铺
- 修法：改用"演示"型或"故事"型叙事，见模板 [showreel-single-flow](../templates/showreel-single-flow.md)
- 严重程度：🟡 严重

**R5.2 · 无 drop 无 outro**
- 表现：视频匀速跑到最后，没有情绪高潮，logo 一扔就结束
- 修法：60-75% 位置必须有 drop 镜；末段必须三段式收束
- 严重程度：🔴 致命

**R5.3 · 首帧太满**
- 表现：第一帧就有 logo、slogan、图形、文字全出场
- 修法：首帧应该"空 + 简"（品牌视频首帧空 → 末帧满是经典结构）
- 严重程度：🟡 严重

### R6 · 素材类反例

**R6.1 · 用户上传的图没进正片**
- 表现：用户传了 logo 和产品图，视频里全是抽象文字模板
- 修法：素材上传时打 "usage" 标签，"必现"的素材必须至少出现一次
- 严重程度：🔴 致命（用户失望的第一来源）

**R6.2 · 上传的图硬塞进视频**
- 表现：用户传了一张与视频风格不符的图，视频里直接用了
- 修法：Agent 判断上传图的调性是否符合方向卡，不符合的降级到"参考"
- 严重程度：🟡 严重

## 使用方式

Agent 生成每一镜后，**必须**遍历本反例库自检：
- 如果命中 🔴 致命反例 → 该镜作废，重新生成
- 如果命中 🟡 严重反例 → 记录警告，用户闸门时提示
