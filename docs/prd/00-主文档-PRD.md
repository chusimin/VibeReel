## C — Context（业务上下文）

- **一句话定位**：给小白把内容一键做成视频的平台，**一套骨架装三类视频**：产品 showreel / 教学小视频 / 知识科普。
- **核心抽象（全平台的魂，AI 必先理解）**：
  - **公共骨架**（三类共用、顺序固定的闸门流）：`方向选择 → 分镜确认 → 分段确认 → 终检`。
  - **四件套**（每个类型的差异，挂在骨架上）：**知识包**（什么算好的判断）/ **风格包**（配色字体动效 + genai 提示词基因）/ **闸门配置**（比标准多/少/不同的闸门）/ **QA 规则**（体检特有项）。
  - 选类型 = 装上那套四件套，再走同一条公共骨架。**四件套四片各自注入哪一层，见 [01-架构-architecture](01-架构-architecture.md) §6.2。**
- **技术栈（摘要，全量见 [01-架构-architecture](01-架构-architecture.md) §8）**：
  - 前后端一体 **Next.js 14 App Router + React 18 + TS(strict) + Tailwind 3**，单台 **Node 20 LTS** 常驻进程。
  - 创意 Agent：`@anthropic-ai/claude-agent-sdk`（headless）。渲染：子进程调用既有 **vibemotion CLI**（复用，**禁止重写**）。
  - **多后端渲染**：每镜按内容选后端（remotion / 生成式 / lottie / 静帧推拉），各自出 mp4 段、统一合成。分镜草稿：**OpenAI gpt-image**。存储：**本地文件系统，无数据库**。
- **Key 策略**：
  - **POC 阶段（当前）**：创意步**不用 API key**——后端子进程 spawn 本机 **`claude` CLI**（`claude -p … --output-format json --model …`），复用本地 Claude Code 登录态。详见 [07-智能体-agent](07-智能体-agent.md) §运行方式。
  - **v1 生产**：**Anthropic key = BYO**（用户设置页填，仅存浏览器 localStorage，请求透传，服务端用完即弃、不落盘）。
  - **OpenAI（gpt-image）+ 生成式视频 key = 平台配置**（服务端 env，**绝不下发前端**）。
  - 账号 / 计费 / 付费墙 = **v2**（作者先配 key 给熟人用）。
- **用户**：作者 + **5–10 位**设计师 / 产品「小白」朋友。**共享密码进入、不开放注册、无账号体系**；单机低并发。
- **参考产品**：交互参考 Runway / Pika / 可灵；成片质感对标 linear.app（showreel）/ 深空图解（科普）/ 亮色 keynote（教学）。

### 核心背景（执行 AI 必读）
1. 本项目把作者已有 skill **「vibe-motion-video」包成网页平台**，**视频生成全部复用 vibemotion CLI**（边界见 [06-引擎边界-engine-integration](06-引擎边界-engine-integration.md)），**禁止重写引擎**。
2. 平台价值 = **多类型抽象**：三类视频共用一条公共骨架，差异全收进四件套，四件套**最大化复用引擎已有资产**（structures / docs/types / styles / audio）。
3. **三类型一起做**，但实现上**逐场景落地**：先把 showreel 全链跑通，再教学、再科普；每个场景独立可验收。
4. **效果靠多后端**：信息镜走 Remotion（精确），氛围镜走生成式（高级感），动图标走 Lottie，素材图走静帧推拉。这是 v1 的核心提质点。

## AR — 架构总览（结构）

> 这是主文档的"地图缩略图"。完整四层职责、技术层内部分工、成片合成详解、状态机转移表、三词汇对照、四件套注入点全在 **[01-架构-architecture](01-架构-architecture.md)**。

**四层，上层只依赖下层：**

```
┌ 用户层（表面功能）─ 用户看得见点得到的 ────────────────────────────┐
│  登录 · 设置 · 新建(选类型/输入/画幅/风格/传素材) · 4 闸门 · 进度 · 下载 │
└──────────────────────────┬─────────────────────────────────────────┘
┌ 技术层（我们写的代码）─ 把用户决策变成视频的中枢 ───────────────────┐
│  API(只触发) · 编排 orchestrator+queue(大脑) ·                       │
│  能力: agent(创意) / fourpack(四件套) / draft(草稿) /                │
│        renderers(4 后端) / tts(配音) / 合成(ffmpeg 拼接+混音+字幕)   │
└──────────────────────────┬─────────────────────────────────────────┘
┌ 引擎层（vibemotion，复用·禁改）─ 真正出像素的老底子 ─────────────────┐
│  remotion 段渲染 · qa 体检 · export 导出                            │
└──────────────────────────┬─────────────────────────────────────────┘
┌ 数据层（文件系统，无数据库）─ 单一真相源 ───────────────────────────┐
│  data/projects/<id>/ : project.json + 引擎产物 + assets/ + drafts/  │
└─────────────────────────────────────────────────────────────────────┘
```

**一条片子的流水线**（🟦创意=agent写JSON，⬜机械=CLI/后端/ffmpeg，🔶闸门=停下等用户）：

```
⬜ingest → 🟦brief → 🟦concept→🔶① → 🟦script→🔶(科普/教学) → 🟦storyboard
  → ⬜draft→🔶② → ⬜voicing → ⬜rendering→🔶③(逐镜) → ⬜assemble → ⬜qa→🔶④ → done
```

闸门序列由四件套的 **gates 配置**决定（showreel 无 script 闸门）。每步"谁执行/产物/转移条件"见 [01-架构-architecture](01-架构-architecture.md) §5 状态机表。

**成片合成 ≠ 一个动作**：不同分镜各出一段，`assemble` 是 `拼接 concat → 混音 mux → 烧字幕 → 导出` 一条 ffmpeg 子流水线；"看起来统一"靠渲染时强制统一 config（拼接救不了拼贴感）。**谁来拼已定：技术层自己用 ffmpeg 拼**（引擎不碰外来段），详见 [01-架构-architecture](01-架构-architecture.md) §4 与 [06-引擎边界-engine-integration](06-引擎边界-engine-integration.md) §2。

## S — Scope（范围）

### v1 必做（MUST）

**平台骨架**
- 共享密码进入；设置页填 **Anthropic key** + 选模型档（Haiku / **Sonnet 4.6 默认** / Opus），存 localStorage。
- 新建项目：**选视频类型**（showreel / 教学 / 科普）→ 填输入（URL 或 想法，二选一）→ **画幅**（**16:9 默认** / 9:16 / 1:1）→ 选风格卡 → **素材上传引导**。
- **公共骨架四闸门**，三类共用、顺序固定：①方向选择 ②分镜确认 ③分段确认 ④终检；类型可按四件套**插入特有闸门或跳过**（如科普在①后插「讲稿/知识点确认」）。
- **逐分镜流式**：每个分镜渲完即展示，不等整片；显示「生成中」态（见 [04-视觉规范-design-system](04-视觉规范-design-system.md) B3）。
- 全程实时进度；项目列表可重开（续跑 / 下载）。

**三类型四件套**（复用引擎为主，缺口补在引擎层）
- showreel：四件套现成，直接接。
- 知识科普：四件套现成（structure `knowledge-explainer`/`knowledge-popsci`、深空图解风格），**含特有闸门「讲稿/知识点确认」**+ 特有 QA「主画面必须承载信息」。
- 教学小视频：**知识包(playbook) + 音频预设需在引擎层新建**（见 [06-引擎边界-engine-integration](06-引擎边界-engine-integration.md) §4）；结构暂复用 `talking-blogger` 调整。

**素材上传引导**（开始前一次性收齐）
- 引导用户把「能传的都传了」：产品截图 / logo / 品牌色 / 字体 / 现成图或片段。
- 素材进项目素材库，供后端取用（尤其静帧推拉、Remotion 背景、生成式参考图）。

**草稿 → 正片（两阶段）**
- 分镜闸门：agent 写完分镜 → 每镜 **gpt-image 出一张草稿图** → 用户看「草稿 + 文字 beat」确认方向。**草稿仅供方向确认，非正片精确长相**（UI 明示）。
- 分段闸门：确认后每镜**按其后端**渲精确正片，逐段流式 👍 / 👎。

**多后端渲染（v1 接 4 个）**：`remotion`（信息镜）/ `generative`（氛围镜）/ `lottie`（动图标）/ `still-kenburns`（素材图推拉）。agent 在分镜阶段给每个 scene 选 `renderer`，render 阶段分发。详见 [05-生成后端-renderers](05-生成后端-renderers.md)。

**配音 / 字幕**：showreel **默认关**；教学 / 科普**默认开**。开则 edge-tts 合成 + 烧字幕；兜底见 G。

**数据**：项目**元数据驱动**，`project.json` 为单一真相源（见 [03-数据模型-data-model](03-数据模型-data-model.md)）；**分镜级版本记录**：每次分镜被改追加一条 revision，可回看（v1 先「可回看」，回滚排 v2）。

### v1 不做（MUST NOT）
- ❌ 账号 / 注册 / OAuth / 付费墙 / 计费 —— 共享密码 + 作者配置 key，付费排 v2。
- ❌ 服务端持久化用户 Anthropic key —— BYO 只透传、用完即弃。
- ❌ 数据库（Postgres / SQLite / Prisma）—— 文件式 `data/projects/<id>/` 够用。
- ❌ Remotion Lambda / 云渲染农场 —— 本机渲染 +（生成式后端走云 API）；规模化排 v2。
- ❌ 在线 JSON / 时间轴编辑器 —— 闸门只给「选 / 批 / 重做 / 看草稿」。
- ❌ 三类型以外（纯口播 / 小红书图文等）—— 先把三类做透。
- ❌ 在 VibeReel 里改 vibemotion 引擎逻辑（除非补四件套缺口，那走引擎仓库）。
- ❌ gsap 录制后端 / 真人录屏后端 —— 排 v2（v1 先 4 个后端）。

### v2 远期（IDEAS）
- 平台托管全部 key + 真账号 + DB + 计费 / 付费墙；云渲染农场（Remotion Lambda + 生成式队列）。
- 更多后端（gsap 录制 / 录屏 / 3D）；更多类型；分镜版本**回滚**；MCP 集成市场；本地桥接。

## G — Guardrails（护栏）

### 禁止事项
- ❌ **不要重写 / 绕过 vibemotion 引擎**自己实现视频生成。引擎是核心资产，本项目只做编排壳 + 多后端分发。
- ❌ **不要引入数据库**。文件式状态是明确决策。
- ❌ **不要加账号 / 注册 / OAuth / 计费 / 支付**（v1）。BYO + 共享密码 + 作者配置 key。
- ❌ **不要把用户 Anthropic key 写入任何文件 / 日志 / 截图 / 服务端持久层**；**不要把平台 key（OpenAI / 生成式）下发到前端 / 客户端 bundle**。
- ❌ **不要让任一后端无视统一 `config`**（分辨率 / fps / safeArea / 色板 / 字体）。跨后端 / 跨类型**视觉统一是硬约束**，拼贴感 = 验收不通过。
- ❌ **不要换技术栈 / 加未列依赖**（不加 shadcn / 状态库等）。要加先问。
- ❌ **不要在闸门里给 JSON / 时间轴编辑器**。小白只做「选 / 批 / 重做 / 看草稿」。
- ❌ **不要把「草稿」当「正片预览」呈现**。草稿必须标「方向参考」。
- ❌ **不要在本仓库改引擎**；教学四件套缺口去 vibe-motion-video 仓库补。
- ❌ **失焦警报**：遇 PRD 字段模糊 / 矛盾 / 标注的「开放问题」→ **停下问作者**，不脑补默认值。
