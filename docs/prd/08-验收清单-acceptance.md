## 公共能力验收

对应三类视频共用的公共骨架（登录 / key 引导 / 四闸门 / 多后端流式 / 元数据驱动），详见 [02-流程与闸门-flows-and-gates](02-流程与闸门-flows-and-gates.md) 与 [05-生成后端-renderers](05-生成后端-renderers.md)。

- [ ] 错误密码进不去；正确密码刷新仍保持登录。
- [ ] 未填 Anthropic key 时新建页黄条提示并能跳设置；填后消失。
- [ ] 三种类型各能新建并跑到方向闸门（≤60s 出 2–3 概念卡）。
- [ ] 画幅选 9:16 时，成片实际为 1080×1920（验证引擎显式收到 `--platform douyin`，**没有因引擎默认而出成 16:9**）。
- [ ] 素材上传后，至少一个 `still-kenburns` 镜用到了上传图。
- [ ] 分镜闸门每镜显示 gpt-image 草稿 + 文字 beat；草稿生成期间有生成态、不空屏。
- [ ] 分段闸门逐镜流式出现，占位卡原位替换成播放器、不整页刷新；某镜走生成式后端、某镜走 remotion，成片画幅 / 色板 / 字体一致（无拼贴感）。
- [ ] 分镜「打回重做」后内容有可见变化，且 `project.json` 里该 scene 多一条 revision。
- [ ] 全通过后出成片，可下载能播放的 mp4（开配音则另有 srt）。
- [ ] 刷新 / SSE 断线重连后进度与闸门不丢；项目列表能重开。

## 分类型验收

对应每类视频挂在公共骨架上的四件套差异（showreel / 科普 / 教学 的闸门配置与 QA 规则），详见 [02-流程与闸门-flows-and-gates](02-流程与闸门-flows-and-gates.md)。

- [ ] **showreel**：默认 16:9、默认不配音；情绪曲线在分镜里可辨（hook→演示→payoff）。
- [ ] **科普**：方向闸门后**先出讲稿/知识点闸门**，可逐条确认 / 打回；终检 QA 含「主画面是否承载信息」项。
- [ ] **教学**：能加载教学知识包跑通（引擎四件套补齐后）；默认配音开。

## 异常验收

对应行为规范里的异常路径（Anthropic / URL 抓取 / gpt-image / 生成式后端 / CLI 步失败的降级与重试），渲染后端相关见 [05-生成后端-renderers](05-生成后端-renderers.md)、引擎调用相关见 [06-引擎边界-engine-integration](06-引擎边界-engine-integration.md)。

- [ ] 错 Anthropic key → 概念阶段红 toast + 重试，进度不丢。
- [ ] 抓不动的 URL → 可改纯文字想法重试成功。
- [ ] gpt-image 失败 → 该镜降级为纯文字 beat，闸门仍可继续。
- [ ] 生成式后端失败 → 该镜可重试或回退，不整链崩。
- [ ] 某 CLI 步非零退出 → 红 toast + stderr 末若干行 + 重试该步。

## 资源验收

对应单机低并发下的长任务队列与渲染并发约束，详见 [06-引擎边界-engine-integration](06-引擎边界-engine-integration.md)。

- [ ] 单机同时跑 2 个项目渲染不崩；第 3 个排队显示「排队中」。
