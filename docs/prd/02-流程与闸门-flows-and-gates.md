本文档把 VibeReel 从「打开网页」到「下载成片」的全过程拆成一连串「触发 → 响应」，每一步都标出异常路径。**公共骨架四闸门**（①方向选择 → ②分镜确认 → ③分段确认 → ④终检）三类视频共用、顺序固定；类型可按四件套**插入特有闸门或跳过**（如科普/教学在①后插「讲稿/知识点确认」）。`Stage` / `GateId` / `GateState` 的字段定义见 [03-数据模型-data-model](03-数据模型-data-model.md)。

## 登录与 key 前置检查

| 触发 | 响应 | 异常 / 分支 |
|---|---|---|
| 用户未登录访问任意页 | 跳登录页 | — |
| 登录页提交密码 | 与 env `APP_PASSWORD` 比对 | 对 → 下发 httpOnly 签名 cookie（7 天，用 `COOKIE_SECRET` 签）→ 进主页；错 → 红字「密码错误」 |
| 进入新建页（**POC**） | 创意步用本机 `claude` CLI，无需 key，直接可建；设置页显示「本地 claude CLI 已检测」 | CLI 不可用（未登录/未安装）→ 提示跑 `claude` 登录 / 装 CLI |
| 进入新建页（**v1**，localStorage 无 Anthropic key） | 顶部黄条「先填 Anthropic key」+ 跳设置页 | 填好 key 后黄条消失 |

## 新建流程

按顺序触发，每步可见，最终落地一个项目并开始 SSE 推进度：

1. **选类型**（showreel / 教学 / 科普）。
2. **加载该类型四件套**：structure + playbook 注入 agent + 候选风格卡 + 闸门配置 + QA 规则。
3. **填输入**（URL 或 想法，二选一）。
4. **选画幅**（16:9 默认 / 9:16 / 1:1）。
5. **选风格卡**。
6. **素材上传**（可跳过）。
7. 点「开始」→ 生成 `projectId = crypto.randomUUID()` → 跳项目页 → SSE 推进度。

## ingest 抓取

| 触发 | 响应 | 异常路径 |
|---|---|---|
| 项目进入 ingest | 显示「抓取内容…」 | URL 抓取失败 → 停步，红 toast +「改纯文字想法重试」 |

## 闸门①方向（GateId: concept）

- **触发**：ingest / briefing 完成，进入 concept 阶段。
- **响应**：展示 **2–3 张概念卡**，每卡含 标题 / 一句话基调 / 关键视觉词 → 用户选一张 →「用这个方向」。

## 讲稿 / 知识点闸门（科普·教学特有，GateId: script）

- **触发**：script-first 类型（科普 / 教学）在分镜前插一关；**showreel 跳过此关**。
- **响应**：展示**逐字讲稿 / 知识点清单**，用户**逐条确认或打回**。
- **缘由（保留原文措辞）**：内容对错用户最有发言权，因此把判断权交给用户逐条确认。

## 闸门②分镜（含 gpt-image 草稿，GateId: storyboard）

- **触发**：方向（及讲稿，如有）确认后，agent 写分镜 → 每镜用 gpt-image 出一张草稿图 → 进入 storyboard 闸门。
- **响应**：展示 **beat 草稿卡列表**（gpt-image 草稿图 + 文字 beat）。两个操作：
  - **「确认分镜」** → 进渲染。
  - **「打回重做」** → 弹一句话意见框（可空）→ agent 重写 → **追加一条 revision** → 重新展示；**内容须有可见变化**。
- **生成态护栏**：草稿生成期间显示生成态，**不空屏**（B3 生成态设计见 [01-架构-architecture](01-架构-architecture.md) 状态机说明，占位卡规格见主文档视觉规范）。
- **草稿定位**：草稿仅供方向确认，UI 必须明示「草稿·方向参考」，不得当正片预览呈现。

## 闸门③分段（多后端流式，GateId: chunk）

- **触发**：分镜确认后，逐分镜按其后端渲精确正片。
- **响应**：**逐分镜渲染，每镜完成 → 原位替换成播放器** +「👍下一段 / 👎重做这段」。
  - **👎重做这段** → 弹一句话意见 → 改该 scene → **仅重渲该镜** → 追加一条 revision。
  - **末镜 👍** → 触发 assemble + qa + export。
- **流式护栏**：占位卡原位替换成播放器，**不跳动、不整页刷新**；跨后端（某镜走生成式、某镜走 remotion）成片画幅 / 色板 / 字体须一致，无拼贴感（分段渲染分发见 [05-生成后端-renderers](05-生成后端-renderers.md)）。

## 闸门④终检（GateId: final）

- **触发**：assemble + qa + export 完成。
- **响应**：成片播放器 + 下载（mp4 / srt / 音频 / zip）。
- **QA 红项处理**：qa 出红项 → 顶部黄条列出，**仍允许下载**（v1 不阻断）。

## 异常通用路径

下列异常**停在当前步、进度不丢**，给出可重试入口，不整链崩：

| 异常来源 | 响应 |
|---|---|
| Anthropic 失败（401 / 超额 / 网络） | 红 toast + 摘要 +「重试」，停在当前步、进度不丢 |
| CLI / 后端非零退出 | 红 toast + stderr 末 20 行 +「重试该步」 |
| gpt-image 草稿失败 | 该镜降级：显示纯文字 beat，闸门仍可继续 |
| 正片生成式后端失败 | 该镜降级：回退 remotion 占位 或 提示换后端，不整链崩 |

## SSE 断线与刷新恢复

- **SSE 断线** → 自动重连，按服务器当前状态重绘（SSE 通道与状态机见 [01-架构-architecture](01-架构-architecture.md)）。
- **刷新项目页** → 从 `project.json` 恢复，不丢进度（`project.json` 字段见 [03-数据模型-data-model](03-数据模型-data-model.md)）。
