## 核心做法

新增统一封面组件 [`web/components/Cover.tsx`](../../web/components/Cover.tsx)：
即使没有真实图，也用「字符串种子 → 双色相」稳定生成影院级渐变静帧，
让每个视觉位都像「高级内容」而不是空盒子；有真实草稿帧/缩略图时叠在其上。
配套 CSS 见 `globals.css` 的 `.cover / .film / .waveform / .stage`。

## 仍存在的差距（数据所限，未编造）

- v1 封面里的**真实产品摄影/UI 截图**：后端目前只产出纯色测试渲染（那些紫/绿/粉块是真实 mp4/草稿帧），
  无真实素材库摄影。v3 用美术化封面 + 真实渲染帧承载，**结构与观感一致，但不是照片**。
- 素材库的**全局素材流**（品牌/截图/视频片段满屏）：后端只有自定义风格 + 角色，无全局聚合接口。
- 终检的**版本记录表**：outputs 无版本历史。

这些需要后端补接口才能 100% 贴稿，已记于 [`docs/设计稿与实现差异-mismatches.md`](../../docs/设计稿与实现差异-mismatches.md)。
