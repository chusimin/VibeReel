# VibeReel · 设计语言 / Design Language

> 从 `design/high-fidelity-ui/01–08` 高保真稿提取的设计语言，作为前端实现的单一视觉真相源。
> **取代** 旧的亮色规范（`docs/prd/04-视觉规范`：亮色高 key + 橙色火花）。本版为 **暗色 · 极简黑白（monochrome）**。
> 维护：改 token 只改这里 + `web/app/globals.css` 的 `:root`，组件用语义类名，不写死颜色。

---

## 1. 基调 Principles

1. **暗色 · 极简黑白**：近黑画布，信息用"亮度层级"分层，而非颜色。颜色只用于**状态语义**。
2. **白即火花**：主操作是**纯白填充按钮**（深色字）。全站没有彩色品牌强调色——克制是高级感的来源。
3. **编辑感留白**：大标题、宽行距、卡片间距充足；密集信息（表格/分镜）也要靠留白呼吸。
4. **一处一焦点**：每屏一个主操作（白按钮）+ 一个右侧"摘要/详情"面板承托上下文。
5. **状态可读优先**："需要你处理"和"系统在忙"必须一眼分得清（见 §6 状态系统——这是高保真稿里需要修正的点）。

## 2. 颜色 Color

### 2.1 表面（亮度层级 / elevation）
| Token | 值 | 用途 |
|---|---|---|
| `--bg` | `#060708` | 应用画布（最底层） |
| `--surface-1` | `#0E0F11` | 卡片 / 面板 / 侧栏 |
| `--surface-2` | `#16181B` | 输入框 / hover / 选中态底 / 嵌套卡 |
| `--surface-3` | `#212327` | chip / 徽章 / 提示块（参考稿"推荐"徽章 ≈ `#202122`） |
| `--overlay` | `#0A0B0D` | 浮层 / 弹窗 / 播放器底 |

### 2.2 描边
| Token | 值 | 用途 |
|---|---|---|
| `--border` | `#1F2226` | 卡片/分割线（克制） |
| `--border-strong` | `#2C2F34` | 输入框/选中卡描边 |
| `--hairline` | `rgba(255,255,255,.06)` | 表格行分割等极细线 |

### 2.3 文字
| Token | 值 | 用途 |
|---|---|---|
| `--text` | `#F3F4F5` | 主文字（近白，不用纯白，降眩光） |
| `--text-2` | `#A2A6AD` | 次要/说明文字 |
| `--text-3` | `#6C7077` | 三级/占位/表头 |
| `--text-disabled` | `#474B51` | 禁用 |

> 对比度（修复审查 #对比度）：正文/次要文字须 ≥ 4.5:1。`--text-2 #A2A6AD` on `--surface-1` ≈ 7:1 ✓；`--text-3` 仅用于非关键标签。

### 2.4 主操作 / 中性按钮
| Token | 值 | 用途 |
|---|---|---|
| `--primary` | `#FFFFFF` | 主按钮底（白填充） |
| `--on-primary` | `#0A0B0D` | 主按钮字（深） |
| `--primary-hover` | `#E9EAEB` | 主按钮 hover |
| 次按钮 | 透明底 + `--border-strong` 描边 + `--text` 字 | ghost |

### 2.5 状态色（仅此处用彩色 · 均为低饱和以融入暗背景）
| Token | 值 | 语义 |
|---|---|---|
| `--st-pending` | `#6C7077`（灰） | 待生成 / 未开始 / 草稿 / 排队 |
| `--st-progress` | `#5B8DEF`（蓝，取自稿 `#5A8CDA`） | **系统在忙**：渲染中 / 进行中 |
| `--st-review` | `#E0A23B`（琥珀） | **需要你处理**：待确认 / 待审 |
| `--st-ok` | `#4FAE7E`（绿，取自稿 `#4E956D`） | 已通过 / 已完成 / 质检通过 |
| `--st-err` | `#E5615E`（红） | 失败 / 需重做 |

> ⚠️ **关键修正**：高保真稿里 `待确认` 和 `渲染中` **都是蓝色**（见审查发现 #1）。本规范把 `待确认` 改为**琥珀色**，与"渲染中（蓝）"区分——"待你处理" ≠ "系统在跑"。

## 3. 字体 Typography
- 字族：`"Inter", system-ui, "PingFang SC", sans-serif`（`next/font` 加载 Inter）。等宽：`ui-monospace`（数字/时长/分辨率）。
- 字号阶梯：

| 用途 | 字号/字重/字距 |
|---|---|
| 页面大标题 | `30px / 600 / -0.01em` |
| 区块标题 | `18px / 600` |
| 卡片标题 | `15px / 600` |
| 正文 | `14px / 400 / line-height 1.55` |
| 次要/说明 | `13px / 400` |
| 标签/表头/徽章 | `12px / 500 / 字距 0.02em`（表头可大写或加字距） |
| 按钮 | `14px / 500` |

---

## 4. 间距 / 圆角 / 阴影
- 间距基准 `4`，常用 `8 / 12 / 16 / 20 / 24 / 32 / 40`。卡片内边距 `20–24`。
- 圆角：卡片 `--r-card:14px`；按钮/输入 `--r-btn:10px`；小卡/缩略 `--r-sm:8px`；pill `--r-pill:999px`。
- 阴影（暗色下克制，靠表面亮度分层；阴影只给浮层）：
  - 卡片：无阴影，靠 `--surface-1` + `--border`。
  - 浮层/弹窗：`--shadow-pop: 0 12px 40px rgba(0,0,0,.55)`。
  - 选中态：`box-shadow: 0 0 0 1px var(--border-strong)`（描边式高亮，不用彩色光晕）。

## 5. 布局 Layout

### 5.1 顶层页面（首页 / 历史 / 素材库 / 设置）
```
┌─────┬────────────────────────────────────────────┐
│     │ 顶栏：搜索框(⌘K) ········· 状态胶囊/操作 通知 头像 │
│ 左  ├────────────────────────────────────────────┤
│ 导  │                                            │
│ 航  │   页面主体（max-width ~1200，居中或左对齐）    │
│ 栏  │                                            │
│     │                              [可选右侧栏]    │
└─────┴────────────────────────────────────────────┘
```
- **左导航栏**（`Sidebar`，宽 ~`76px` 图标轨）：顶部 Logo `V`；中部图标+短标签 `首页/历史/素材库/设置`，当前项高亮（`--surface-2` 底 + 白图标）；底部"创作空间/用户"块。
- **顶栏**（`56–60px`）：左侧搜索框（`⌘K`），右侧：本地引擎状态胶囊、主操作（如"新建视频 +"）、通知铃、头像。
- 右侧栏（如"本周概览"、素材详情）：宽 ~`300px`，`--surface-1` 卡。

### 5.2 工作流页面
- **/new（新建）**：顶部"← 返回首页" + Logo + 引擎状态；主体为编号表单分区 + 右侧"生成摘要"贴边面板；**底部水平步进器**（内容准备→方向确认→分镜确认→分段预览→最终检查）。
- **/projects/[id]（方向/分镜/分段/终检）**：**左侧竖向步骤栏**（5 步，已完成✓/进行中高亮/未开始灰）+ 底部用户块；中部大预览 + 列表；右侧"详情/摘要"面板 + 主操作。

### 5.3 内容宽度
- 顶层主体最大宽 `1200px`；工作流媒体区可满宽；表单类 `max-width 720–960`。

## 6. 状态系统 Status（统一词表 + 颜色——修复审查 #1/#5）

> 修复：高保真稿状态词过多且"完成态"重复（草稿/已确认/已通过/已完成…）、`待确认` 与 `渲染中` 同蓝。下面是**统一后的最小词表**，全站复用。

### 6.1 镜头/分段级（对应后端 `SceneStatus`）
| 后端值 | 展示词 | 颜色 | 圆点/图标 |
|---|---|---|---|
| `pending` | 待生成 | `--st-pending` 灰 | ● 实心灰 |
| `rendering` | 渲染中 (NN%) | `--st-progress` 蓝 | ◌ 旋转环 |
| `await_review` | 待确认 | `--st-review` 琥珀 | ● 琥珀（"需你处理"） |
| `approved` | 已通过 | `--st-ok` 绿 | ✓ 绿 |
| `redo` | 待重做 | `--st-err` 红 | ↺ 红 |

### 6.2 项目级（首页/历史 `状态` 列）
| 语义 | 展示词 | 颜色 |
|---|---|---|
| 早期闸门未渲染 | 草稿 | 灰 |
| 正在渲染 | 渲染中 | 蓝 |
| 已出终片 | 已完成 | 绿 |
| 归档 | 已归档 | 灰（更暗） |

### 6.3 通用 `Status` 组件
- 形态：`● 文案`（圆点 6px + 文案 13px）。**不靠颜色单独承载语义**——永远带文案（无障碍）。
- 渲染中带百分比时显示旋转环 + `NN%`。

## 7. 组件 Components

- **按钮**：主=白填充深字（`生成方向/确认方向/通过/下载视频/用于新视频`）；次=透明描边；危险=红字描边；尺寸 `sm/md`。hover 轻微变亮，active 下沉 1px。
- **输入/选择**：`--surface-2` 底 + `--border-strong` 描边；focus 加亮描边（**不用橙色光晕**，用 `--text-3→--border-strong` 提亮 + 极淡白 ring `0 0 0 3px rgba(255,255,255,.06)`）。
- **可选大卡**（类型卡/方向卡/风格卡）：`--surface-1` + `--border`；hover 描边变亮；选中 = `--border-strong` 描边 + 左上角选中标。**带"推荐"徽章**（`--surface-3` 胶囊）。
- **Pill 切换**（画幅/筛选 tab）：未选 = `--text-2` 描边胶囊；选中 = 白字 + `--surface-2` 底 + `--border-strong`。
- **Tabs**（内容来源/素材库分类）：底部 1px 指示线高亮当前。
- **表格**：表头 `--text-3` 12px；行用 `--hairline` 分割；`状态`列用 §6 组件；`操作`列按上下文给动词（继续/预览/导出/开始渲染/取消）。行 hover `--surface-2`。
- **步进器（stepper/rail）**：圆形序号；已完成=✓+绿描边；当前=白底深字；未开始=`--border` 描边灰字。**修复审查 #4**：工作流主步进器用①②③；表单内分区**不要**再用同款①②③，改用小标题或字母，避免双重编号打架。
- **缩略图条 + 列表（分镜/分段）**：**修复审查 #2**——二者不并列重复。缩略图条只做**横向导航+选中**（带状态点）；详细信息（画面/时长/状态/操作）放**可切换的列表视图**，默认其一，用"列表/网格"切换，不同时全量展示同一份镜头表。
- **右侧摘要/详情面板**：键值对列表（图标+标签+值）；**修复审查 #3**——若展示"草稿图/配音/字幕"等可变设置，要么是**可点切换的开关**，要么明确为只读且在表单里有对应控件，不能出现"看得到改不了"。
- **媒体预览/播放器**：`--overlay` 底，居中播放按钮（白半透圆），底部渐变信息条（时间/时长）。
- **拖拽上传区**：虚线 `--border-strong` 框 + 居中说明"可直接拖入…支持图片/视频/模型/SVG"。
- **空/错/载状态**（补审查 #6）：空态给插画位+一句引导+主操作；渲染失败给红占位卡 + 原因 + "重试该镜"；加载用 shimmer 骨架卡（不整页刷新）。

## 8. 动效 Motion
- 时长 `120–250ms`，缓动 `ease`/`cubic-bezier(.2,.7,.2,1)`。
- hover/选中只做亮度与描边过渡；进入用 8px 上浮淡入。
- 渲染中：骨架 shimmer + 旋转环；渲完原位替换为播放器，**不跳动、不整页刷新**。
- 进度条带 `NN%` 与（可选）剩余时间 ETA（补审查 #渲染缺 ETA）。

---

## 9. 实现约定
- 所有颜色经 `:root` CSS 变量；组件用语义类名（`.btn/.card/.pill/.rail/.status` 等），页面 JSX 不写死 hex。
- Tailwind `theme.colors` 仅做语义别名映射到变量，避免双套色板。
- 暗色为唯一主题（不做亮色切换）。
